## 三、是否持久化？

### 3.1 持久化机制

**答案**: ✅ **100%持久化**

**存储后端**: SQLite数据库（默认）

**数据库文件**: `data/.memory/icme.db`

**表结构**: `memories` 表

```sql
CREATE TABLE IF NOT EXISTS memories (
    id TEXT PRIMARY KEY,                -- 记忆ID
    content TEXT NOT NULL,              -- 内容全文
    content_segmented TEXT NOT NULL DEFAULT '',  -- 分词内容（FTS5）
    layer TEXT NOT NULL,                -- 记忆层（sensory/working/...）
    tags TEXT NOT NULL DEFAULT '[]',    -- 标签列表
    priority TEXT NOT NULL DEFAULT 'medium',  -- 优先级
    value_score REAL NOT NULL DEFAULT 0.5,  -- 价值分数
    access_count INTEGER NOT NULL DEFAULT 0,  -- 访问次数
    created_at REAL NOT NULL,           -- 创建时间
    last_accessed REAL NOT NULL,        -- 最后访问时间
    size_bytes INTEGER NOT NULL,        -- 大小（字节）
    metadata TEXT NOT NULL DEFAULT '{}',  -- 元数据
    asset_id TEXT,                      -- 资产ID（关联数字资产）
    related_ids TEXT NOT NULL DEFAULT '[]',  -- 关联ID
    changelog TEXT NOT NULL DEFAULT '[]',  -- 变更日志
    quality_score REAL NOT NULL DEFAULT 0.5  -- 质量分数
);
```

### 3.2 持久化特性

| 特性 | 说明 | 优势 |
|------|------|------|
| **WAL模式** | Write-Ahead Logging | 高并发读写 |
| **FTS5索引** | 全文搜索索引 | 零配置语义搜索 |
| **B-tree索引** | 层级索引 | O(log n)查询 |
| **事务批量写入** | 批量提交 | 100x性能提升 |
| **自动VACUUM** | 空间回收 | 空间优化 |
| **跨平台兼容** | SQLite3内置 | 无依赖 |

### 3.3 持久化流程

## 四、与数字资产的关系

### 4.1 三重绑定机制

**核心关系**: `memory_id ↔ asset_id ↔ content_hash`

```
记忆条目 (memories表)
  ├── memory_id: "c6655d380d796018"
  ├── content: "对话全文..."
  ├── asset_id: "sensory:a1b2c3d4:0001"  ← 关联数字资产
  └── metadata: {...}

数字资产 (asset_registry表)
  ├── asset_id: "sensory:a1b2c3d4:0001"
  ├── memory_id: "c6655d380d796018"      ← 关联记忆条目
  ├── content_hash: "a1b2c3d4e5f6..."    ← 内容哈希
  ├── version: 1                         ← 版本号
  ├── parent_version_id: ""              ← 父版本ID
  └── status: "active"                    ← 资产状态
```

### 4.2 自动注册流程

**文件**: `core/hybrid_engine.py` 第448-495行

```python
if self._asset_registry and entry_id:
    try:
        from .asset_atom import AssetAtom, ContentType, Provenance

        # 1. 计算内容哈希
        content_hash = self._asset_registry.compute_content_hash(content)

        # 2. 判断内容类型
        content_type = ContentType.UNKNOWN
        src = (metadata or {}).get("source", "")
        if "trae_capture" in src or "conversation" in src:
            content_type = ContentType.CONVERSATION  # 对话类型
        elif "file" in src or "snapshot" in src:
            content_type = ContentType.FILE          # 文件类型
        elif "decision" in src or actual_layer == "episodic":
            content_type = ContentType.DECISION      # 决策类型
        elif actual_layer == "semantic":
            content_type = ContentType.KNOWLEDGE     # 知识类型
        elif actual_layer == "meta":
            content_type = ContentType.RULE          # 规则类型

        # 3. 创建资产原子
        atom = AssetAtom(
            memory_id=entry_id,
            layer=actual_layer,
            content_type=content_type,
            content_hash=content_hash,
            provenance=Provenance(
                created_by="hybrid_engine",
                created_at=time.time(),
                reason="Auto-registered from remember()",
                session_id=(metadata or {}).get("session_id", ""),
            ),
        )

        # 4. 注册资产（自动生成asset_id）
        tcl_ids = (metadata or {}).get("tcl_canonical_ids", [])
        asset_id = self._asset_registry.register(atom, content=content, tcl_ids=tcl_ids)

        # 5. 返回asset_id
        result["asset_id"] = asset_id
```

### 4.3 资产类型映射

| L0 Sensory来源 | content_type | 说明 |
|----------------|--------------|------|
| trae_capture | **CONVERSATION** | 对话内容 |
| file snapshot | FILE | 文件快照 |
| decision | DECISION | 决策记录 |
| - | UNKNOWN | 未知类型 |

### 4.4 版本链管理

**策略D混合存储**: 每次remember()自动生成带版本链的内容快照

```
版本链示例:

  ├── parent_version_id: ""

asset_id: "sensory:a1b2c3d4:0002"  (版本2)
  ├── parent_version_id: "sensory:a1b2c3d4:0001"
  └── status: "active"  (当前活跃)

asset_id: "sensory:a1b2c3d4:0003"  (版本3)
  ├── parent_version_id: "sensory:a1b2c3d4:0002"
  └── status: "active"  (当前活跃)
```

## 六、数据完整性保证

### 6.1 三重绑定验证

```python
# 验证脚本
def verify_triple_binding(memory_id: str):
    # 1. 查询记忆条目
    memory = db.query("SELECT * FROM memories WHERE id=?", (memory_id,))

    # 2. 查询数字资产
    asset = db.query("SELECT * FROM asset_registry WHERE memory_id=?", (memory_id,))

    # 3. 验证三重绑定
    assert memory["id"] == asset["memory_id"], "memory_id不匹配"
    assert memory["asset_id"] == asset["asset_id"], "asset_id不匹配"
    assert asset["content_hash"] == compute_hash(memory["content"]), "content_hash不匹配"

    print("✅ 三重绑定验证通过")
```

### 6.2 版本链验证

```python
def verify_version_chain(asset_id: str):
    # 1. 查询资产
    asset = db.query("SELECT * FROM asset_registry WHERE asset_id=?", (asset_id,))

    # 2. 递归验证父版本
    if asset["parent_version_id"]:
        parent = db.query("SELECT * FROM asset_registry WHERE asset_id=?",
                         (asset["parent_version_id"],))
        assert parent["status"] == "superseded", "父版本未标记为superseded"
        verify_version_chain(parent["asset_id"])

    print("✅ 版本链验证通过")
```
