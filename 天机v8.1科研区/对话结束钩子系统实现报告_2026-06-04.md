## 二、核心功能

### 2.1 自动触发机制

**触发链路**:

```
对话结束 → on_conversation_end() → ConversationHookManager.trigger_by_platform()
    ↓
TraeConversationHook.trigger() / QoderConversationHook.trigger()
    ↓
调用天机API: POST /api/active/capture_conversation
    ↓
写入L0 Sensory + L1 Working + L3 Episodic
    ↓
返回捕获结果
```

### 2.2 容错机制

- **钩子失败不影响主流程**: 每个钩子独立执行，失败不影响其他钩子
- **超时控制**: API调用超时5秒，避免阻塞
- **异常捕获**: 所有异常都被捕获并记录，不会抛出到上层
- **启用/禁用**: 支持动态启用/禁用钩子

### 2.3 统计监控

每个钩子和管理器都提供统计信息：


## 三、使用指南

### 3.1 在Trae IDE中集成

**方法1: 直接调用入口函数**

```python
from active_memory.conversation_hook import on_conversation_end

# 对话结束时调用
result = on_conversation_end(
    user_input="用户的问题",
    ai_response="AI的回答",
    session_id="session-123",
    agent_id="lingxi",
    platform="trae",
    tags=["auto-capture"],
    mcp_calls=["memory_remember", "memory_recall"],
    file_operations=[{"path": "test.py", "action": "write"}]
)

if result.get("success"):
    print(f"✅ 对话已自动录入: turn_id={result['turn_id']}")
else:
    print(f"❌ 录入失败: {result.get('error')}")
```

**方法2: 作为对话结束回调**

```python
# 在Trae IDE的对话结束事件中添加

    on_conversation_end(
        user_input=user_input,
        ai_response=ai_response,
        session_id=session_id,
        platform="trae"
    )
```

### 3.3 高级用法

**动态启用/禁用钩子**:

```python
from active_memory.conversation_hook import get_hook_manager

manager = get_hook_manager()

# 禁用所有钩子
manager.disable_all()

# 启用所有钩子
manager.enable_all()

# 查看统计信息
stats = manager.get_stats()
print(f"总对话数: {stats['total_conversations']}")
print(f"成功率: {stats['total_success'] / stats['total_hooks_triggered'] * 100:.1f}%")
```

**自定义钩子**:

```python
from active_memory.conversation_hook import (
    ConversationEndHook,
    ConversationContext,
    get_hook_manager
)

class CustomHook(ConversationEndHook):
    def trigger(self, context: ConversationContext):
        # 自定义逻辑
        print(f"自定义钩子触发: {context.session_id}")
        return {"success": True, "custom": "data"}

# 注册自定义钩子
manager = get_hook_manager()
manager.register(CustomHook())
```
