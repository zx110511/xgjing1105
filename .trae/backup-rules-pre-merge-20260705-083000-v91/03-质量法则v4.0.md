# 03-质量法则 (Quality Laws) v4.0

**优先级**: P0-强制 | **激活**: 始终生效 | **适用**: 全Agent

---

## 一、代码质量红线 (不可妥协)

| 指标 | 红线 | 检测方式 |
|------|------|---------|
| 类型注解率 | >=80% | mypy/pyright |
| 测试覆盖率 | >=80% | pytest-cov/c8 |
| 圈复杂度 | <=10 | radon/eslint |
| 函数长度 | <=50行 | lint规则 |
| 安全漏洞 | 0 Critical | npm audit/bandit |
| 非法路径引用 | 0处 | scripts/path_audit.py |

### 路径引用唯一性法则 (PATH-LAW-001)
- **唯一权威源**: `core/config.py` 的 `PYTHON_EXECUTABLE` + `get_python_executable()`
- **禁止**: 硬编码绝对路径 / 非项目内置Python路径
- **必须**: `from core.config import get_python_executable` / `sys.executable` fallback

## 二、灵魂拷问标准 (P0强制)

**核心问题**: "真的能用吗？" — 不是"代码写了"，而是"代码真的能跑通、能闭环、能支撑真实场景"

### 六维验证体系

| 维度 | 权重 | 核心问题 | 通过标准 |
|------|------|---------|---------|
| 可实现性 | 1/6 | 技术上真的能实现吗？ | 依赖可用+接口可达+环境兼容 |
| 真实场景支撑 | 1/6 | 真实场景真的能支撑吗？ | 主流程+异常流程+边界场景全覆盖 |
| 并发稳定 | 1/6 | 并发下真的稳定吗？ | 无竞态+无死锁+无内存泄漏 |
| 跨模块集成 | 1/6 | 跨模块真的能集成吗？ | 接口契约+数据格式+错误传播 |
| 兼容性 | 1/6 | 兼容性真的没问题吗？ | 向后兼容+跨平台+版本共存 |
| 边界条件 | 1/6 | 边界条件真的处理了吗？ | 空值+溢出+超时+极端输入 |

**评分门槛**: 六维加权平均 >= 9.95分 (满分10分)

## 三、Stage Gate门禁 (强制)

```
SG-0: 环境准备 → Python 3.12+ / Node 24+ / 天机healthy / MCP连接正常
SG-1: Import+类型 → 无循环依赖 / mypy--strict / ruff check
SG-2: 功能验证 → 覆盖率>=80% / 集成测试 / 边界+错误路径
SG-3: MCP集成 → 工具调用>=95% / CRUD闭环 / TVP格式正确
SG-4: 回归验证 → 全量通过 / 无新增失败 / 性能无退化(<5%)
```

**Gate不通过 = 流程终止**，必须修复后重新提交。

| **辅助工具** | tianji_summarize | 智能摘要 | P2 | 长文本处理 |

### MCP调用规范 (强制)

**调用前必须声明**:
```
[TVP-MCP] 🔌 MCP调用:
├─ 调用者: @{agent}
├─ 服务器: {mcp_server}
├─ 工具: {tool_name}
├─ 参数摘要: {50字}
└─ 超时设置: {timeout_seconds}s
```

**调用后必须验证**:
- 返回状态: success/error
- 返回数据: 符合预期格式
- 调用延迟: <SLA阈值
- 错误处理: 记录到L3 Episodic

### MCP参数验证 (新增)

**memory_remember参数验证**:
```
必填: content, layer
选填: tags[], priority, metadata{}
层级: sensory|working|short_term|episodic|semantic|meta
验证:
  - content长度>=30字符 (min_content_length)
  - layer必须是6层之一
  - tags至少2个
  - priority必须是low|medium|high|critical
```

**memory_recall参数验证**:
```
必填: query
选填: layers[], limit, threshold
策略: semantic(语义) > exact(精确) > hybrid(混合)
验证:
  - query长度>=5字符
  - layers必须是6层之一或数组
  - limit必须>=1且<=100
  - threshold必须>=0且<=1
```

### MCP调用频率限制 (新增)

| 工具 | 单Agent频率限制 | 系统总频率限制 | 超限处理 |
|------|----------------|---------------|---------|
| memory_remember | 10次/分钟 | 100次/分钟 | 队列暂存 |

### MCP错误分级响应 (新增)

| 级别 | 响应时限 | 必须动作 |
|------|---------|---------|
| Critical (MCP Server崩溃) | <30s | 停止操作 + L3+L5写入 + 告警 + 降级 |

- 每次用户对话后
- 对话历史审计
- Agent输出质量评估

### 执行流程
1. **对话捕获**：trae_capture → 完整对话上下文

```yaml

  - 边界条件考虑
efficiency: # 效率
  - 工具调用次数
  - 对话轮次
safety: # 安全性
  - 无敏感信息泄露
  - 无危险操作
```

- 代码/文档写入前
- 内容规范检查
- 发布前审查

### 执行流程
1. **内容读取**：获取待审查内容
2. **规范匹配**：对比编码规范/文档规范

```yaml
code_review:

  - Markdown: 标题层级, 链接有效
```

## 七、强制记忆记录义务 (强壮)

### 三条核心规则

**规则1: 对话必记录**
- 每次用户消息 → L0 Sensory自动缓存
- 每次对话结束 → L1 Working或L3 Episodic持久化

**规则2: 结构化数据格式**
- 必填: type, timestamp, session_id, user_input, agent, response_summary, success
- 标签: 至少2个 (自动生成+手动补充)

**规则3: 异常必记录**
- 错误记录必须存入L3 Episodic (强制)
- 必须包含: 错误信息 + 根因分析 + 教训提取
- 错误记录永不删除 (retain_forever=true)
