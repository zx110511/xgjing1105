## 六大根本原则 (强壮化)

### 原则1: 记忆优先决策 (强壮)
- 非平凡决策必须查询天机 (L4知识+L3经验+L5约束)
- 豁免: 仅限简单配置选择 (complexity=low)
- **新增**: 决策前必须调用memory_recall，决策后必须调用memory_remember

### 原则2: 操作必记录 (强壮)
- 写操作前 -> L1 Working (意图+参数)
- 操作后 -> L3 Episodic (结果+影响)
- 系统变更 -> L5 Meta (策略归档)
- **新增**: 每次操作必须有完整的TVP声明和记忆记录

### 原则3: 故障必回溯 (强壮)
- 即时 -> L0捕获原始错误
- 根因 -> L3记录分析结果
- 教训 -> L4沉淀避免方案
- **新增**: 故障后必须触发进化反思环，更新L5 Meta策略

### 原则4: 破除闭门造车 (强壮)
- 推演深度>=6分: **必须**联网搜索资源 + 搜索天机记忆资源
- 推演深度>=4分: **必须**提取天机记忆用作参考
- 资源充足前提下方可执行任务，禁止凭空推演
- **新增**: 资源搜索结果必须记录到L3 Episodic，供后续复用

### 原则5: 灵魂拷问通用化 (强壮)
- 每个功能实现必须通过灵魂级拷问: "真的能用吗？"
- 不是"代码写了"，而是"代码真的能跑通、能闭环、能支撑真实场景"
- 必须有真实可运行的验证，而非仅代码审查
- **新增**: 六维评分>=9.95分，必须基于真实运行数据验证

### 原则6: 对话自动审计 (强壮)
- 每次对话结束前，必须自动化完成对话记录审计
- 最重要审计项: 是否有完整记录对话 + 文件内容
- 审计不通过则补充记录，确保信息不丢失
- **新增**: 审计结果必须写入L5 Meta，触发系统自进化

## 第10条: 规则淘汰机制 (铁律) — 新增

**核心论断**: 规则升级后，旧版规则文件必须自动化淘汰删除，否则违反"极限精简"原则。

### 淘汰标准

| 淘汰条件 | 判断标准 | 执行动作 |
|---------|---------|---------|
| 版本升级 | 新版文件存在（如v6.0.md存在） | 自动删除旧版文件（如v5.0.md） |
| 功能合并 | 多个旧版文件合并为新版 | 自动删除所有旧版文件 |
| 规则废弃 | 规则内容完全废弃 | 自动删除废弃规则文件 |
| 冗余文件 | 同一规则存在多个版本 | 保留最新版，删除其余版本 |

### 淘汰流程 (自动化)

```
Step 1: 识别 → 检测新版规则文件是否存在
Step 2: 列表 → 列出所有旧版规则文件（版本号低于新版）
Step 3: 备份 → 将旧版文件备份到归档目录（可选）
Step 4: 删除 → 自动删除旧版规则文件
Step 5: 记录 → 记录淘汰操作到L3 Episodic + L5 Meta
Step 6: 验证 → 验证规则目录仅保留最新版文件
```

### 淘汰执行时机

| 触发条件 | 执行时机 | 执行方式 |
|---------|---------|---------|
| 规则升级完成 | 升级后立即执行 | 自动化删除 |
| 规则合并完成 | 合并后立即执行 | 自动化删除 |
| 规则废弃确认 | 废弃确认后立即执行 | 自动化删除 |
| 定期清理 | 每30天执行一次 | 自动化扫描+删除 |

### 淘汰保护机制

**禁止删除**:
- v9.1运行基线的原有规则文件（v9.1是运行基线，需要保持稳定）
- 正在使用的规则文件（project_rules.md等入口文件）
- 归档目录中的规则文件（归档=快照，禁止修改）

**必须保留**:
- 最新版规则文件（如v6.0.md）
- 规则入口文件（project_rules.md）
- v9.1原有规则文件（运行基线）

### 淘汰记录机制

每次淘汰操作必须记录：
- 淘汰文件列表（文件名+版本号）
- 淘汰原因（版本升级/功能合并/规则废弃）
- 淘汰时间（ISO8601格式）
- 淘汰结果（成功/失败）
- 记录到L3 Episodic（事件经历）+ L5 Meta（系统策略）

### 淘汰自动化脚本

```python
# 伪代码示例
def auto_retire_rules(new_version):
    # Step 1: 识别新版文件
    new_files = detect_new_version_files(new_version)
    
    # Step 2: 列出旧版文件
    old_files = list_old_version_files(new_version)
    
    # Step 3: 备份（可选）
    if backup_enabled:
        backup_to_archive(old_files)
    
    # Step 4: 删除
    for file in old_files:
        if not is_protected(file):
            delete_file(file)
    
    # Step 5: 记录
    record_retirement(old_files, new_version)
    
    # Step 6: 验证
    verify_only_latest_files()
```

### 淘汰预警机制

**预警触发条件**:
- 规则目录存在多个版本文件
- 旧版文件超过30天未删除
- 淘汰操作未记录到天机

**预警格式**:
