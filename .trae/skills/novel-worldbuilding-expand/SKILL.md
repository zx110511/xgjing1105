# 世界观扩展 (novel/worldbuilding-expand)

## 目的
基于现有世界观设定进行深度扩展，完善力量体系、社会结构、历史背景等。

## 触发场景
- 用户要求"扩展设定"、"丰富世界"、"增加细节"
- @planner Agent执行架构设计任务时
- 发现设定空白或不完整时

## 扩展维度框架

### 维度1: 力量体系深化
- [ ] 能力等级划分细化
- [ ] 获取/修炼方式补充
- [ ] 限制与代价机制
- [ ] 特殊能力变体设计
- [ ] 战斗/应用场景示例

### 维度2: 世界架构扩展
- [ ] 地理环境详图 (地形/气候/资源)
- [ ] 政治/势力格局
- [ ] 经济体系运作
- [ ] 历史/传说背景
- [ ] 文化/宗教习俗

### 维度3: 社会生态构建
- [ ] 种族/族群分类
- [ ] 社会阶层结构
- [ ] 教育/培养体系
- [ ] 法律/道德规范
- [ ] 日常生活方式

### 维度4: 技术水平设定
- [ ] 科技树发展程度
- [ ] 特殊技术/发明
- [ ] 物品/装备体系
- [ ] 交通/通讯方式
- [ ] 医疗/生命科学

## 执行流程

### Phase 1: 现状分析
1. 加载现有设定 (`memory_recall` → "世界观设定")
2. 识别已完善的领域
3. 定位空白/模糊区域
4. 优先级排序

### Phase 2: 扩展设计
1. 选择目标扩展维度
2. 设计具体内容 (遵循内部一致性)
3. 多方案对比选优
4. 与现有设定的融合验证

### Phase 3: 质量验证
1. 自检: 是否引入矛盾
2. 自检: 是否过于复杂
3. 自检: 是否有足够的故事潜力
4. 标记版本号

### Phase 4: 入库存储
1. 调用 `memory_remember` 存储到 semantic 层
2. 更新版本追踪记录
3. 通知相关Agent (@writer/@reviewer)

## 输出格式
```markdown
## {扩展维度} v{version}

### 核心设定
[详细描述...]

### 子项目清单
1. **{子项1}**: [说明]
2. **{子项2}**: [说明]
...

### 与现有设定的关系
- 关联: {已有设定ID}
- 冲突: 无 ✓
- 影响范围: {受影响的章节/情节}

### 故事潜力点
- 可用于: {潜在情节}
- 冲突种子: {可能的矛盾点}
---

*创建时间: {timestamp} | 版本: v{version} | 状态: 待审核*
```

## 设计原则
1. **一致性优先**: 新设定不能与已有设定矛盾
2. **适度留白**: 不必一次性完善所有细节
3. **故事驱动**: 设定应为情节服务，而非堆砌
4. **可演进性**: 预留未来扩展空间

## 绑定Agent
@planner (主要) | @version-keeper (版本管理)

## 协作伙伴
@writer (使用新设定) | @reviewer (审核一致性) | @editor (规划整合)

## 注意事项
- ⚠️ 重大设定变更需先通知@editor和@version-keeper
- ⚠️ 已被引用的历史设定修改需特别谨慎
- ✅ 建议采用"增量迭代"而非"推倒重来"
