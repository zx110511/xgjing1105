# 经验自动沉淀机制架构设计 (Phase 3.4)

**版本**: v1.0
**状态**: 架构设计
**目标**: 替代技能文件中的"最佳实践"，实现经验自动积累与复用

---

## 一、设计背景

### 问题
技能文件中的"最佳实践"是静态的、手写的、容易过时的。
需要一种机制让经验随着使用自动积累、验证、演化。

### 核心思想
> **每一次工具使用都是一次经验采样，每一次成功/失败都是一次经验验证。**

---

## 二、整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    经验自动沉淀闭环                               │
│                                                                 │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐  │
│  │  经验采集 │ →  │  经验评估 │ →  │  经验沉淀 │ →  │  经验复用 │  │
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘  │
│       ↑                                              │         │
│       └──────────────── 反馈循环 ─────────────────────┘         │
└─────────────────────────────────────────────────────────────────┘
```

### 四层架构

| 层级 | 名称 | 职责 | 对应记忆层 |
|------|------|------|-----------|
| L1 | 经验采集层 | 捕获每次工具调用的上下文、参数、结果 | L1 Working |
| L2 | 经验评估层 | 评估成功/失败、质量评分、异常检测 | L2 Short-Term |
| L3 | 经验沉淀层 | 去重、聚类、模式提取、写入经验库 | L3 Episodic |
| L4 | 经验复用层 | 语义检索、推荐、适配、验证 | L4 Semantic |

## 三、经验数据模型

### 经验条目 (Experience Entry)

```python
ExperienceEntry = {
    # 身份
    "experience_id": "exp-{domain}-{timestamp}-{hash}",
    "version": "1.0",
    
    # 领域分类
    "domain": "memory|agent_dispatch|code_review|novel_creation|...",
    "pattern_type": "success_pattern|failure_lesson|best_practice|optimization",
    
    # 触发上下文
    "trigger_context": {
        "task_type": "任务类型",
        "agent": "执行Agent",
        "tool_calls": ["调用的工具链"],
        "input_summary": "输入摘要",
    },
    
    # 解决方案
    "solution": {
        "steps": ["执行步骤"],
        "parameters": {"关键参数"},
        "tool_chain": ["工具调用顺序"],
    },
    
    # 结果评估
    "outcome": {
        "success": True/False,
        "quality_score": 0.0-1.0,
        "duration_ms": 执行耗时,
        "error_type": "错误类型(如果失败)",
    },
    
    # 元数据
    "metadata": {
        "source": "auto_extracted|human_curated",
        "confidence": 0.0-1.0,
        "reuse_count": 0,
        "success_rate": 0.0,
        "first_seen": "ISO8601",
        "last_used": "ISO8601",
        "tags": ["标签列表"],
    },
}
```

### 经验质量分级

| 等级 | 置信度 | 来源 | 复用方式 |
|------|--------|------|---------|
| S级 | >0.95 | 人工策展 + 验证100+次 | 直接推荐，最佳实践 |
| A级 | 0.8-0.95 | 自动提取 + 验证10+次 | 高优先级推荐 |
| B级 | 0.6-0.8 | 自动提取 + 验证3+次 | 可参考，需验证 |
| C级 | 0.4-0.6 | 自动提取 + 验证1-2次 | 低优先级，需谨慎 |
| D级 | <0.4 | 单次成功/失败 | 仅归档，不推荐 |

## 四、核心模块设计

### 4.1 经验采集器 (ExperienceCollector)

**职责**: 无侵入式捕获每次工具调用的完整上下文

**触发时机**:
- 每次MCP工具调用前后
- 每次Agent切换时
- 每次任务完成时

**采集内容**:
```python
collect_context = {
    "task": "当前任务描述",
    "agent": "当前Agent",
    "tool": "调用的工具名",
    "parameters": {"输入参数"},
    "result": {"返回结果"},
    "success": True/False,
    "duration_ms": 耗时,
    "error": "错误信息(如果有)",
    "context_tags": ["上下文标签"],
}
```

**实现方式**:
- MCP工具调用拦截器（类似tianji_intercept）
- 对性能影响 < 1ms（异步写入）
- 失败不影响主流程

### 4.2 经验评估器 (ExperienceEvaluator)

**职责**: 评估每次操作的质量，判断是否值得沉淀为经验

**评估维度**:

| 维度 | 评分方法 | 权重 |
|------|---------|------|
| 任务完成度 | 是否达成目标 | 40% |
| 效率 | 工具调用次数/耗时 | 20% |
| 创新性 | 是否有新方法/新思路 | 15% |
| 可复用性 | 是否通用可迁移 | 15% |
| 稳定性 | 是否可重复成功 | 10% |

**评估触发**:
- 任务成功 → 提取成功模式
- 任务失败 → 提取失败教训
- 异常恢复 → 提取修复经验

### 4.3 经验沉淀器 (ExperienceConsolidator)

**职责**: 将原始采集数据加工为可复用经验

**处理流程**:
```
Step 1: 去重
  - 相似度>0.9的经验合并
  - 保留质量评分更高的版本
  
Step 2: 聚类
  - 按领域+模式类型聚类
  - 识别高频成功模式
  - 识别常见失败模式
  
Step 3: 抽象
  - 从具体案例中提取通用步骤
  - 参数化、模板化
  - 标注适用边界
  
Step 4: 验证
  - 与历史经验对比一致性
  - 计算置信度
  - 升级/降级经验等级
```

**存储位置**: L4 Semantic层 `experience_{domain}` 命名空间

### 4.4 经验复用器 (ExperienceRetriever)

**职责**: 在需要时智能推荐相关经验

**触发场景**:
- 决策前 (agent_dispatch 前)
- 工具调用前 (memory_recall 时自动关联)
- 错误恢复时
- 方案生成时

**检索策略**:
```
Step 1: 语义匹配
  - 根据任务描述语义搜索经验库
  
Step 2: 上下文适配
  - 过滤不适用当前上下文的经验
  - 按当前Agent能力调整推荐
  
Step 3: 质量排序
  - S级 > A级 > B级
  - 成功率高优先
  - 近期使用优先
  
Step 4: 适配建议
  - 标注经验与当前任务的差异
  - 提供适配建议
```

## 五、与技能文件的对比

| 维度 | 技能文件 | 经验自动沉淀 |
|------|---------|-------------|
| 生成方式 | 人工编写 | 自动提取+验证 |
| 更新频率 | 静态，不定期 | 实时，随使用增长 |
| 覆盖范围 | 已知最佳实践 | 成功+失败+优化 |
| 置信度 | 假设正确 | 基于数据验证 |
| 上下文适配 | 通用描述 | 智能适配当前场景 |
| 进化能力 | 人工更新 | 自进化 |
| 维护成本 | 高 | 低 |

## 七、关键技术点

### 7.1 性能保证
- 采集异步化，对主流程影响 < 1ms
- 经验评估后台批量执行（每30分钟）
- 经验检索走向量索引，< 50ms

### 7.2 数据质量
- 经验三元验证法: 成功次数 + 失败对比 + 人工标注
- 置信度衰减: 长期未验证的经验自动降权
- 经验冲突检测: 相互矛盾的经验自动标记

### 7.3 安全与隐私
- 敏感信息自动脱敏（PII/token/密码）
- 经验访问控制（Agent级权限）
- 经验溯源可审计

**灵魂拷问**:
- "经验自动沉淀真的能替代技能文件吗？" — 短期不能完全替代，但长期可以超越，因为它有数据验证和自进化
- "MVP阶段有价值吗？" — 有。即使只是自动记录操作轨迹，也比没有强，而且为后续进化打基础
- "风险是什么？" — 最大风险是低质量经验污染知识库，需要置信度机制和人工审核兜底
