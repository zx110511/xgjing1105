# 天机v9.1深度嵌入Trae — 智能体智能调度最优架构 - Product Requirement Document

## Overview

* **Summary**: 本PRD定义天机v9.1与Trae IDE深度融合的智能体智能调度最优架构。通过自动化提取Trae自定义智能体面板的27个自定义Agent、2个内置Agent、天机v9.1的24个Agent矩阵、6个MCP服务器42个工具的完整配置，设计三层调度架构（Trae面板层→天枢编排层→天机内核层），实现智能体的智能路由、上下文无缝传递、TVP全链路透明、记忆驱动决策的深度嵌入体系。

* **Purpose**: 解决当前天机v9.1 Agent与Trae自定义智能体面板割裂、调度不透明、上下文断裂、记忆不同步的问题，构建从用户交互到内核执行的全链路智能调度闭环。

* **Target Users**: 天机v9.1系统开发者、Trae IDE深度用户、元初系统运维团队

## Goals

* **G1: 全景映射** — 建立Trae面板智能体↔天机Agent矩阵↔MCP工具的完整映射关系，消除信息孤岛

* **G2: 三层调度** — 设计Trae面板层→天枢编排层→天机内核层的三级智能调度架构，实现最优路由

* **G3: TVP全链路** — 实现Agent切换、技能调用、记忆操作、MCP调用的四维TVP声明全链路覆盖

* **G4: 记忆驱动** — 所有非平凡决策必须检索天机L3+L4+L5记忆，操作必记录，形成记忆闭环

* **G5: 无感切换** — 用户在Trae面板切换智能体时，上下文、记忆、状态无缝传递，无感知断裂

* **G6: 自进化闭环** — 调度数据自动沉淀为经验，驱动调度策略持续优化

## Non-Goals (Out of Scope)

* 不修改Trae IDE的核心代码（仅通过配置和MCP扩展）

* 不重构天机v9.1的记忆内核（端口8771保持不变）

* 不开发新的MCP服务器（基于现有6个服务器扩展）

* 不涉及v10.0.1的开发（本项目仅针对v9.1深度嵌入）

* 不做移动端适配（仅桌面端Trae IDE）

**4. 当前问题**

* **割裂问题**: Trae面板智能体与天机Agent矩阵是两套体系，未打通

* **调度黑盒**: Agent切换不透明，用户不知道为什么调用了哪个Agent

* **上下文断裂**: 切换Agent时对话上下文丢失，需重新说明需求

* **记忆不同步**: Trae面板操作不写入天机记忆系统，经验无法沉淀

* **工具冗余**: 多个Agent重复配置相同MCP工具，缺乏统一管控

## Functional Requirements

### FR-1: 智能体全景映射引擎

* 自动发现并注册Trae面板所有自定义智能体

* 建立Trae智能体↔天机Agent的映射关系表

* 支持一对一、一对多、多对一的映射模式

* 自动检测映射冲突并提供修复建议

* 映射关系持久化到天机L4 Semantic层

### FR-2: 三层智能调度架构

* **L1-Trae面板层**: 用户交互入口，负责意图初判和Agent选择

* **L2-天枢编排层**: 核心调度器，负责任务分解、Agent编排、TVP声明

* **L3-天机内核层**: 执行层，31个已注册Agent(24个天机系列+7个Trae官方系列) + 6个MCP服务器 + 记忆系统

* 支持串行/并行/层级/工业化/事件驱动/进化闭环6种协作模式

* 智能路由算法：根据任务特征自动选择最优Agent组合

### FR-3: TVP全链路透明协议

* TVP-Agent: 每次Agent切换必声明（当前→目标+任务+状态+数据流）

* TVP-SKILL: 每次技能调用必声明（调用者+技能+目标Agent+参数+预期）

* TVP-MEM: 每次跨层记忆操作必声明（操作者+操作+层级+内容摘要+关联Agent）

* TVP-MCP: 每次MCP工具调用必声明（调用者+服务器+工具+参数+超时）

* TVP声明实时显示在Trae面板中，用户可随时查看调度链路

### FR-4: 上下文无缝传递

* 切换Agent时自动传递对话上下文（最近N轮）

* 自动提取关键实体、意图、进度状态并传递

* 支持上下文快照和恢复（断点续传）

* 灵犀Agent实时监控上下文完整性，异常自动修复

### FR-5: 记忆驱动决策闭环

* 非平凡决策前自动调用memory\_recall检索L3+L4+L5

* 操作后自动调用memory\_remember写入对应层级

* 系统变更自动写入L5 Meta层

* 故障自动触发进化反思环，根因分析写入L3+L4

* 调度经验自动沉淀为L4知识，驱动未来调度优化

### FR-6: 智能体健康监控

* 实时监控24个天机Agent的健康状态

* Agent失败自动重试、熔断、降级

* 性能指标采集：响应时间、成功率、工具调用数

* 千里Agent负责全局监控，异常自动告警

### FR-7: 配置同步与一致性

* 天机Agent配置变更自动同步到Trae面板

* Trae面板自定义智能体配置自动同步到天机注册表

* 双向同步冲突检测与解决机制

* 配置版本管理（史官Agent负责）

## Non-Functional Requirements

### NFR-1: 性能

* Agent切换延迟 < 500ms（不含LLM调用时间）

* TVP声明生成 < 100ms

* 记忆检索 < 2s（语义搜索）

* 调度决策 < 1s（单步调度）

* 系统总并发支持20个Agent同时运行

### NFR-2: 可靠性

* 核心调度链路可用性 > 99.9%

* Agent失败自动重试最多3次

* 熔断恢复时间 < 120s

* 数据一致性：记忆写入最终一致，延迟 < 10s

### NFR-3: 安全性

* 所有MCP工具调用必须经过权限验证

* 禁止越级调用（严格遵守权限矩阵）

* 敏感操作需人工确认

* 记忆数据永不物理删除（仅软删除/归档）

### NFR-4: 可扩展性

* 新增Agent只需配置，无需修改调度核心

* 新增MCP服务器自动注册和发现

* 调度策略可插拔替换

* 支持水平扩展（多实例部署）

### NFR-5: 可观测性

* 全链路追踪（Trace ID贯穿始终）

* 结构化日志（JSON格式，UTF-8-SIG编码）

* 实时监控仪表盘（千里Agent提供）

* 调度链路可视化（TVP声明组成完整链路图）

## Constraints

### 技术约束

* **前端**: Trae IDE自定义智能体框架（不可修改核心）

* **后端**: Python 3.12 + FastAPI + uvicorn（现有技术栈）

* **记忆**: SQLite + FTS5 + 向量检索（天机v9.1现有架构）

* **通信**: MCP协议 + HTTP REST（端口8771）

* **配置**: JSON格式，UTF-8-SIG编码

* **禁止**: 修改Trae IDE核心代码、修改v9.1记忆内核结构

### 业务约束

* 必须保持天机v9.1现有功能完全兼容

* 必须遵守天机宪法v6.0和智能体法则v4.0

* 调度必须100%透明可追溯（TVP协议）

* 六维验证评分必须 >= 9.95分

### 依赖

* Trae IDE >= 2.3.4（支持自定义智能体）

* 天机v9.1健康运行（端口8771可用）

* 6个MCP服务器全部正常连接

* Python 3.12+ 环境

## Assumptions

* Trae IDE的自定义智能体配置格式保持稳定

* 用户已授权天机v9.1访问Trae配置目录

* MCP协议版本兼容（当前版本可用）

* 天机v9.1的记忆系统性能满足调度查询需求

* 用户理解并接受TVP声明带来的额外信息输出

## Acceptance Criteria

### AC-1: 智能体全景映射完整

* **Given**: 天机v9.1正常运行，Trae IDE已配置自定义智能体

* **When**: 启动映射引擎扫描所有智能体配置

* **Then**: 所有24个天机Agent + 7个非天机Trae自定义Agent + 2个内置Agent全部被发现并建立映射关系

* **Verification**: programmatic

* **Notes**: 映射关系存储在\_AGENT\_REGISTRY.json和天机L4 Semantic层

### AC-2: 三层调度架构正常工作

* **Given**: 用户在Trae面板发起复杂任务（需要多Agent协作）

* **When**: 天枢编排层接收到任务

* **Then**: 自动分解任务→选择最优Agent组合→按协作模式执行→TVP全链路声明→结果聚合返回

* **Verification**: programmatic + human-judgment

* **Notes**: 至少支持6种协作模式中的4种

### AC-3: TVP全链路透明

* **Given**: 任务执行过程中发生Agent切换、工具调用、记忆操作

* **When**: 查看调度链路

* **Then**: 每次切换/调用/操作都有对应的TVP声明，包含完整的四维信息

* **Verification**: programmatic

* **Notes**: TVP声明覆盖率100%，无遗漏

### AC-4: 上下文无缝传递

* **Given**: 用户在对话中途切换到另一个Agent

* **When**: 新Agent开始响应

* **Then**: 新Agent知道之前的对话内容、关键实体、当前进度，无需用户重复说明

* **Verification**: human-judgment + programmatic

* **Notes**: 上下文传递准确率 >= 95%

### AC-5: 记忆驱动决策闭环

* **Given**: 执行非平凡决策任务

* **When**: 决策前后检查记忆操作记录

* **Then**: 决策前有memory\_recall，决策后有memory\_remember，系统变更写入L5 Meta

* **Verification**: programmatic

* **Notes**: 记忆调用覆盖率100%（非平凡决策）

### AC-6: 性能达标

* **Given**: 系统正常负载下

* **When**: 执行标准调度测试用例

* **Then**: Agent切换延迟<500ms，记忆检索<2s，调度决策<1s

* **Verification**: programmatic

* **Notes**: 性能测试基于标准硬件环境

### AC-7: 故障自动恢复

* **Given**: 某个Agent执行失败

* **When**: 触发熔断机制

* **Then**: 自动重试→降级→告警，用户感知最小化

* **Verification**: programmatic

* **Notes**: 重试最多3次，熔断恢复<120s
