## 第一章：哲学根基 — 道域法则级体系的本体论

```
═══════════════════════════════════════════════════════════════════════════
              天机道域 — 从道到108圆满的法则级体系
═══════════════════════════════════════════════════════════════════════════

 道 (TAO) — 天机的本质：AI记忆系统的全部可能性空间
 │
 └─ 一 (ONE) = 闭环进化道
    │  法则(定义应然) × 度量(测量实然) × 演化(驱动转化) = ∞自进化
    │
    ├─ 三 (THREE) = 三才治理模型
    │    体·法则域 (阴·定义) — 天机的宪法：什么必须被遵守
    │    用·度量域 (阳·测量) — 天机的仪表：什么正在发生
    │    枢·演化域 (度·转化) — 天机的引擎：如何从实然到应然
    │
    ├─ 九道 (NINE DAOS) — 三才各三分，九道统摄全局
    │   道一·记忆体道 | 道二·认知体道 | 道三·进化体道
    │   道四·质量体道 | 道五·编排体道 | 道六·容器体道
    │   道七·守护体道 | 道八·知识体道 | 道九·配置体道
    │
    ├─ 三十六地煞 (36 DISHA) — 9道×4术 = 36技术组件
    │   地煞者，地之根基也。属阴，主"术"（技术实现）。
    │   每一个地煞是一个具体的技术组件，承载系统运转的物质基础。
    │
    └─ 七十二天罡 (72 TIANGANG) — 9道×8法 = 72功能能力
       天罡者，天之纲维也。属阳，主"法"（功能原则）。
       每一个天罡是一个抽象的功能能力，指引系统的行为方向。

═══════════════════════════════════════════════════════════════════════════

三才治理闭环：
  法则(应然) ──定义──→ 度量(实然) ──测量──→ 演化(转化) ──驱动──→ 法则(更新)
       ↑                                                           │
       └─────────────────── 自进化闭环 ───────────────────────────┘

三才模型的本质优势：
  1. 法则不是静态规则，而是可演化的活法
  2. 度量不是一次性评分，而是持续感知的活仪表
  3. 演化不是被动变更，而是法则驱动的活引擎
  4. 三者形成自进化闭环，系统永远在"应然→实然→转化→新应然"中进化

三才与DIT决策智能理论的数学对齐：
  GateScore(a) = max(w_i × φ_i(a))  ←→  法则柱的法则执行器
  Conflict(a,b) = Corr(U_a, U_b)    ←→  度量柱的冲突检测
  RSG = Required - Actual            ←→  演化柱的差异诊断
  x(t+1) = x(t) + α[obs(t)-x(t)]   ←→  治理闭环的Lyapunov稳定性

═══════════════════════════════════════════════════════════════════════════
```

## 第四章：道域治理引擎 (DDGE) — 三才治理模型

### 4.1 设计哲学

```
超越"规划+评价+审计"的三才治理模型：

传统模型:  规划(一次性) → 评价(定期) → 审计(事后)
  问题: 线性、被动、断裂、无自进化能力

三才模型:  法则(持续定义) ⇄ 度量(持续感知) ⇄ 演化(持续转化)
  优势: 闭环、主动、连续、自进化

三才关系:
  法则 = 度量的目标 (没有法则，度量无方向)
  度量 = 演化的依据 (没有度量，演化无依据)
  演化 = 法则的更新 (没有演化，法则无生命)

三才闭环:
  法则定义"应然" → 度量测量"实然" → 演化驱动"转化" → 法则更新"新应然"
  ↑                                                         │
  └─────────────────── 自进化闭环 ──────────────────────────┘
```

#### 4.5.2 动态评分引擎 (ScoreEngine) (v2.0: 增加边界条件)

```
评分公式:
  ModuleScore(t) = Σ(w_i × D_i(t)) × TrendFactor(t) × LawCompliance(t)

约束条件:
  1. Σw_i = 1.0 (权重归一化)
  2. 0 ≤ D_i(t) ≤ 100 (维度分数有界)
  3. TrendFactor(t) ∈ [0.8, 1.2] (趋势因子有界，防止发散)
  4. LawCompliance(t) ∈ [0, 1] (合规率有界)
  5. ModuleScore(t) ∈ [0, 100] (总分有界)

边界条件:
  - 当LawCompliance=0时，ModuleScore=0 (零合规=零分)
  - 当所有D_i=0时，ModuleScore=0 (零维度=零分)
  - 当TrendFactor<0.9持续7天，触发METRIC_ALERT事件
  - 当LawCompliance<0.5，触发LAW_UPDATE事件(法则过严检查)

动态权重示例:
  道一·记忆体道: FUNC=0.25, QUAL=0.20, TEST=0.15, DOC=0.05, PERF=0.15, ARCH=0.10, SEC=0.05, EVOL=0.05
  道七·守护体道: FUNC=0.15, QUAL=0.15, TEST=0.20, DOC=0.05, PERF=0.15, ARCH=0.05, SEC=0.20, EVOL=0.05
```

### 4.6 枢·演化柱 (EVOLVE PILLAR)

#### 4.6.1 差异诊断器 (GapDiagnoser)

```
差异 = 法则定义的"应然" - 度量测量的"实然"

差异类型:
  1. 法则差距: 法则要求但未实现 (如: TVP声明未持久化)
  2. 度量差距: 度量分数低于目标 (如: 测试覆盖率<80%)
  3. 演化差距: 进化潜力不足 (如: 技术债务比>25%)
  4. 一致性差距: 地煞间接口不一致 (如: 事件格式不统一)

差异优先级:
  P0-CRITICAL: 影响系统核心功能 (如: 记忆写入失败)
  P1-HIGH: 影响系统质量/安全 (如: 安全漏洞)
  P2-MEDIUM: 影响系统效率/可维护性 (如: 技术债务)
  P3-LOW: 影响系统体验/美观 (如: 文档不完整)
```

#### 4.6.2 演化规划器 (EvoPlanner)

```
规划原则:
  1. 止血优先: P0差距立即修复
  2. 道域均衡: 九道差距均衡缩小，不偏科
  3. 因果驱动: 每个演化必须关联因果对
  4. 渐进安全: 大变更拆分为小步骤，每步可回滚
```

#### 4.6.3 演化执行器 (EvoExecutor) (v2.0: 增加失败恢复)

```
执行模式:
  1. 自动执行: L1参数调优 (安全边界内自动调整)
  2. 半自动: L2规则增补 (LLM提案+人工审核)
  3. 人工审核: L3架构演化 (全面评审+灰度发布)

执行保障:
  - 每步演化前: 快照当前状态
  - 每步演化后: 度量柱验证效果
  - 效果不达标: 自动回滚到快照
  - 效果达标: 记录到L3经验层+更新法则

失败恢复机制 (v2.0新增):
  - 快照损坏: 从天机L3 Episodic层恢复最近有效快照
  - 回滚失败: 进入安全模式(仅法则柱运行，度量/演化柱暂停)
  - 连续3次失败: 触发LAW_UPDATE事件，降级为人工审核模式
  - 安全模式恢复: 逐级恢复(法则→度量→演化)，每级观察5分钟

执行闭环:
```

## 第七章：道域治理引擎核心代码设计 (v2.0: 增加异常处理和降级策略)

### 7.1 治理引擎主类

```python
class DaoDomainGovernanceEngine:
    """
    道域治理引擎 (DDGE) — 三才治理模型的主入口

    三柱:
      体·法则柱 (LawPillar) — 定义应然
      用·度量柱 (MetricPillar) — 测量实然
      枢·演化柱 (EvolvePillar) — 驱动转化

    闭环: 法则→度量→演化→法则(更新)

    降级策略 (v2.0):
      - 法则柱不可用: 仅度量+演化(只读模式)
      - 度量柱不可用: 仅法则(静态合规模式)
      - 演化柱不可用: 仅法则+度量(监控模式)
      - 总线不可用: 三柱独立运行(降级模式)
    """

    def __init__(self, memory_api_url: str = "http://127.0.0.1:8771"):
        self.law_pillar = LawPillar(memory_api_url)
        self.metric_pillar = MetricPillar(memory_api_url)
        self.evolve_pillar = EvolvePillar(memory_api_url)
        self.governance_bus = GovernanceBus()
        self._degraded_mode = False
        self._wire_pillars()

    def _wire_pillars(self):
        try:
            self.governance_bus.subscribe("LAW_UPDATE", self.metric_pillar.on_law_update)
            self.governance_bus.subscribe("LAW_UPDATE", self.evolve_pillar.on_law_update)
            self.governance_bus.subscribe("METRIC_ALERT", self.law_pillar.on_metric_alert)
            self.governance_bus.subscribe("METRIC_ALERT", self.evolve_pillar.on_metric_alert)
            self.governance_bus.subscribe("EVOLVE_RESULT", self.law_pillar.on_evolve_result)
            self.governance_bus.subscribe("EVOLVE_RESULT", self.metric_pillar.on_evolve_result)
            self.governance_bus.subscribe("EVOLVE_REQUEST", self.law_pillar.on_evolve_request)
        except Exception as e:
            self._degraded_mode = True
            self._log_degradation(f"总线接线失败: {e}")

    def govern(self, context: Dict) -> GovernanceResult:
        """执行一次完整的治理循环"""
        try:
            law_state = self.law_pillar.get_current_state()
        except Exception as e:
            return GovernanceResult(status="degraded", reason=f"法则柱异常: {e}")

        try:
            metric_state = self.metric_pillar.measure(law_state)
        except Exception as e:
            metric_state = MetricState(status="unavailable", reason=str(e))

        try:
            gap_report = self.evolve_pillar.diagnose(law_state, metric_state)
            evolve_plan = self.evolve_pillar.plan(gap_report)
        except Exception as e:
            gap_report = GapReport(status="unavailable")
            evolve_plan = EvolvePlan(status="unavailable", reason=str(e))

        return GovernanceResult(
            law_state=law_state,
            metric_state=metric_state,
            gap_report=gap_report,
            evolve_plan=evolve_plan,
            degraded=self._degraded_mode,
        )

    def execute_evolution(self, step: EvolveStep) -> EvolveResult:
        """执行一个演化步骤"""
        snapshot = self._create_snapshot()
        if snapshot is None:
            return EvolveResult(status="failed", reason="快照创建失败")

        try:
            result = self.evolve_pillar.execute(step)
        except Exception as e:
            self._rollback(snapshot)
            return EvolveResult(status="failed", reason=f"执行异常: {e}")

        verification = self.metric_pillar.verify_evolution(step, result)
        if not verification.passed:
            rollback_ok = self._rollback(snapshot)
            if not rollback_ok:
                self._enter_safe_mode()
            return EvolveResult(status="rolled_back", reason=verification.reason)

        self.law_pillar.record_evolution(step, result)
        return result

    def _create_snapshot(self) -> Optional[SystemSnapshot]:
        try:
            return SystemSnapshot.capture()
        except Exception:
            return None

    def _rollback(self, snapshot: SystemSnapshot) -> bool:
        try:
            return snapshot.restore()
        except Exception:
            return False

    def _enter_safe_mode(self):
        self._degraded_mode = True
        self.metric_pillar.pause()
        self.evolve_pillar.pause()

    def _log_degradation(self, reason: str):
        pass
```
