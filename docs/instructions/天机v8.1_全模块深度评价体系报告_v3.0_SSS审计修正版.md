### 1.4 技术债务影响评分(DIS)模型

```python
DIS = w1 × CodeComplexity + w2 × DeliveryFriction + w3 × IncidentCorrelation

其中:
- CodeComplexity: 圈复杂度均值 / 圈复杂度阈值(10) → 归一化[0,1]
- DeliveryFriction: (实际开发时间 - 估算时间) / 估算时间 → [0,∞], 截断至[0,1]
- IncidentCorrelation: 模块相关故障数 / 总故障数 → [0,1]
  ⚠️ v3.0标注: 此指标当前无法计算(天机v9.1无故障追踪系统)
  替代方案: 使用代码审查中发现的Bug密度作为代理指标

权重: w1=0.4, w2=0.35, w3=0.25

风险等级:
- DIS < 0.3: 低风险 (绿色)
- 0.3 ≤ DIS < 0.6: 中风险 (黄色)
- DIS ≥ 0.6: 高风险 (红色, 需立即处理)
```
