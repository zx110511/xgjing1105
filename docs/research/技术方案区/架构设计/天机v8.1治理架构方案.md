## 四、启动流程时序

```
1. tianji_launcher.py 加载
   ├─ APP_DIR / LOG_DIR 初始化
   └─ 基础日志记录器就绪

2. GovernanceOrchestrator 实例化
   ├─ 设置 sys.path
   └─ 初始化状态字典 + 锁

3. governor.bootstrap()
   ├─ Phase 2a: 扫描 core/*.py → AST解析 → 注册到Registry
   ├─ Phase 2b: analyzer.analyze(core_dir) → StaticAnalysisReport
   ├─ Phase 2c: pipeline.govern_module() → 逐个创建 GovernanceRecord
   └─ Phase 2d: 生成 AuditReport → 保存JSON到 logs/audit_reports/

4. start_tray()
   ├─ TianjiTray 创建系统托盘图标
   └─ 治理菜单项就绪

5. tianji_service.py 后台启动
   └─ FastAPI/WebSocket 服务
```

## 五、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 治理组件位置 | launcher内嵌 | 单文件部署，避免循环导入 |
| 导入策略 | 延迟导入(try/except) | 治理降级不影响托盘基础功能 |
| 状态同步 | 线程安全Lock | 托盘菜单可随时调用 |
| 审计等级 | SSS级(72项) | 确保治理闭环 |
| 错误策略 | 阶段隔离 | 任一阶段失败不影响前面阶段的成果 |
