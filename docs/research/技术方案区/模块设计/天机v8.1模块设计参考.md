## 设计原则

1. **单向依赖**: 高层可依赖低层，低层不可依赖高层
2. **接口隔离**: 每个模块暴露有限的 MethodSignature
3. **能力声明**: 通过 capabilities 字段声明公开能力
4. **反能力声明**: anti_responsibilities 明确禁止行为
5. **状态生命周期**: REGISTERED → ACTIVE → DEGRADED → ERROR → DECOMMISSIONED

## 模块注册规范

每个模块必须声明：
- `module_id`: 唯一标识（文件名stem）
- `module_version`: 版本号 "9.0.0"
- `tier`: 所属层级（ModuleTier枚举）
- `module_type`: 模块类型（ModuleType枚举）
- `domain`: 业务领域
- `dependencies`: 所依赖的其他模块（ModuleDependency列表）
- `public_api`: 对外暴露的方法签名（MethodSignature列表）
