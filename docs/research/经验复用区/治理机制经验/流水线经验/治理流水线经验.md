## 流水线架构

```
GovernancePipeline
│
├── Phase 1: 规划 (Plan)
│   ├── 模块定义声明
│   ├── 依赖关系声明
│   └── API设计审查
│       ↓ Gate: 定义完整性检查
│
├── Phase 2: 审计 (Audit)
│   ├── 静态代码分析
│   ├── 合规性校验
│   └── 循环依赖检测
│       ↓ Gate: 审计通过检查
│
├── Phase 3: 落地 (Implement)
│   ├── 模块注册到Registry
│   ├── 生命周期状态转换
│   └── 依赖关系绑定
│       ↓ Gate: 注册成功检查
│
└── Phase 4: 审批 (Approve)
    ├── 验证结果汇总
    ├── 审计记录生成
    └── 签准（自动/铁卫/镇山/天枢）
```

## 核心阶段枚举

### PipelinePhase
| 枚举值 | 中文 | 职责 |
|--------|------|------|
| PLAN | 规划 | 定义模块结构、声明依赖 |
| AUDIT | 审计 | 静态分析、合规检查 |
| IMPLEMENT | 落地 | 注册到Registry、状态转换 |
| APPROVE | 审批 | 结果验证、审计存档 |

### PhaseStatus
| 枚举值 | 含义 |
|--------|------|
| PENDING | 等待执行 |
| IN_PROGRESS | 执行中 |
| PASSED | 已通过 |
| FAILED | 已失败 |
| BLOCKED | 被阻塞 |
| SKIPPED | 已跳过 |

### AuditVerdict
| 枚举值 | 含义 |
|--------|------|
| PASS | 通过 |
| FAIL | 失败 |
| CONDITIONAL_PASS | 条件通过 |
| NEEDS_REVIEW | 需要人工审核 |

### ApprovalLevel
| 枚举值 | 审批方 | 说明 |
|--------|--------|------|
| AUTO | 自动 | 无问题自动通过 |
| TIEWEI_L0 | 铁卫L0 | 质量门禁审批 |
| ZHENSHAN_L4 | 镇山L4 | 架构层审批 |
| TIANSHU | 天枢 | 最高级审批 |

## Phase2集成经验

### 流水线启动流程
```python
# GovernanceOrchestrator._run_pipeline_phase()
for module in registry.list_all():
    record = pipeline.govern_module(
        pipeline.create_governance_pipeline(),
        module,
        analysis_report
    )
    # 状态同步回Registry
    registry.update_lifecycle_state(
        module.module_id,
        derive_state_from_record(record)
    )
```

### 关键教训

1. **生命周期状态须用枚举**：禁止 `"active"` 字符串，必须用 `ModuleLifecycleState.ACTIVE`
2. **状态同步是双向的**：Pipeline → Registry + Registry → Pipeline
3. **审批级别按严重程度升级**：AUTO → TIEWEI → ZHENSHAN → TIANSHU
4. **每个Gate都要有明确的准入/准出条件**

### 防重复注册
```python
# 检查模块是否已存在
existing = registry.get(module_id)
if existing:
    pipeline.skip(module_id, reason="already_registered")
    continue
```
