## 分析器架构

```
StaticDependencyAnalyzer
├── analyze(source_dir) → StaticAnalysisReport
│   ├── AST扫描 → 提取 ImportDependency
│   ├── 依赖图构建 → Dict[source_module, List[target_module]]
│   ├── 循环依赖检测 → List[List[str]]
│   ├── 验证规则执行 → List[ValidationFinding]
│   └── 报告生成 → StaticAnalysisReport
│
└── 验证规则集
    ├── R001: 语法错误检测
    ├── R002: 循环依赖检测
    ├── R003: 命名规范验证
    └── R004: 层级约束验证
```

## 核心数据结构

### ImportDependency
```python
@dataclass
class ImportDependency:
    source_module: str      # 源模块名
    target_module: str      # 目标模块名
    import_type: str        # "absolute" | "relative" | "from_import"
    imported_names: List[str]
    line_number: int
```

### ValidationFinding
```python
@dataclass
class ValidationFinding:
    rule_id: str            # "R001" 格式
    rule_name: str          # 规则名称
    severity: ValidationSeverity  # ERROR | WARNING | INFO
    module_id: str          # 问题所在模块
    message: str            # 简短描述
    detail: str             # 详细信息
    line_number: int        # 问题行号
    suggestion: str         # 修复建议
```

### StaticAnalysisReport
```python
@dataclass
class StaticAnalysisReport:
    analyzed_at: float                          # 分析时间戳
    total_files: int                            # 扫描文件数
    total_modules: int                          # 分析模块数
    total_imports: int                          # 导入总数
    total_classes: int                          # 类总数
    total_functions: int                        # 函数总数
    dependencies: List[ImportDependency]        # 所有依赖
    dependency_graph: Dict[str, List[str]]      # 依赖图
    circular_dependencies: List[List[str]]      # 循环依赖
    findings: List[ValidationFinding]           # 发现列表
    summary: Dict[str, int]                     # 汇总统计
```

## 模块层级约束

| 层级 | 枚举值 | 说明 | 依赖规则 |
|------|--------|------|----------|
| L0 | 质量门禁 | quality_gate, enforcement_hook | 可依赖任何层 |
| L1 | 感知层 | sensory modules | 不可依赖L2/L3/L4 |
| L2 | 认知层 | cognitive modules | 不可依赖L3/L4 |
| L3 | 存档层 | archival modules | 不可依赖L4 |
| L4 | 系统层 | system modules | 仅可依赖L4 |

## Phase2集成经验

### 数据闭环
```
StaticAnalyzer.analyze(core_dir)
    → StaticAnalysisReport
    → sync_analyzer_to_registry(analyzer, registry, report)
        → registry.archive_findings(findings)
        → registry.update_dependencies(dependencies)
```

### 关键教训

1. **使用公共API analyze()**：不要尝试调用 `_create_import_visitor` 等私有方法
2. **report.dependencies必须显式赋值**：报告对象不会自动填充所有字段
3. **circular_dependencies 不是 circular_deps**：字段名需对照源文件确认
4. **依赖图的key是模块名（不含.py）**：`__init__` 节点需要特殊处理

### 分析命令
```python
from core.static_analyzer import StaticDependencyAnalyzer
analyzer = StaticDependencyAnalyzer()
report = analyzer.analyze("core/")
print(f"模块: {report.total_modules}, 依赖: {len(report.dependencies)}, 循环: {report.circular_dependencies}")
```
