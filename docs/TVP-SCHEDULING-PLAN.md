## 🎯 调度原则

### 1. TVP透明调度协议

- ✅ **每次Agent切换必须声明身份和工作状态**
- ✅ **100%可视化协作过程**
- ✅ **消除单智能体假象**

### 2. 调度模式选择

| 模式 | 适用场景 | 示例 |
|------|---------|------|
| **串行流水线** | 有严格依赖顺序的任务 | 开发→审查→测试 |
| **并行分析** | 多维度独立分析 | 性能分析+健康检查 |
| **层级分解** | 大型复杂项目 | 模块化开发 |

### 3. 异常处理与降级

```
[TVP] 🎯 异常处理流程:

发现异常
    ↓
[TVP] 🎯 @orchestrator → 评估异常等级
    ↓
├─ P0 (严重): 立即停止 → @test-verifier → 修复 → 重新调度
├─ P1 (重要): 记录问题 → 继续执行 → 后续修复
└─ P2 (一般): 记录问题 → 后续优化
```
